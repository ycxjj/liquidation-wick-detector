名词解释

资金池与偿付
偿付能力比率
资金池 ÷ 总保额 × 100%。反映基金能否覆盖潜在赔付：≥60% 为 healthy；约 45%–60% 为 watch（你当前 50%）；更低则进入 danger，可能暂停新赔。
状态 healthy / watch / danger
精算状态机档位。watch 表示需关注：24h 全局限额收紧、单笔赔付可能按偿付比例折减。
资金池 / 保额
环境变量 WICKSHIELD_POOL 与 WICKSHIELD_COVERAGE（示例 50000 / 100000），用于偿付检查与压力测试，不是链上实时余额。
赔付额度（每日 00:00 重置）
今日已赔付
本自然日已通过 monitor 累计入账的理赔金额（USDT）。0 表示今日尚未发生批准赔付。
24h 上限 / 剩余赔付额度
全站单日赔付熔断上限（watch 档常见 3000 U）及剩余额度 = 上限 − 今日已赔付。用尽后新索赔会被拒绝，防止透支。
单人上限
单笔索赔最高可赔金额（watch 档常见 300 U），与动态封顶、振幅比例共同约束最终赔付。
24h 赔付额度使用
进度条 = 今日已赔付占 24h 上限的百分比。0% 表示当日熔断未触发。
币种定价与风险档
风险档 A-/A/A+ 与 B/C/D
A 档按定价系数再细分：A-（系数 1~5，如 BTC/XAU/XAUT）、A（5~10，如 DASH/INJ）、A+（10~25，如 RIVER/ORDI）。系数 = 有效日费率÷冷启动底价，并叠加峰值振幅与插针频率。B/C/D 仍为原字母档。
动态附加费（当月）
全站当月已批准赔付笔数越多，新保单费率乘子越高（0 笔=1.0×，1 笔≈1.08×，递增至 20+ 笔≈1.7×）。与 A 档细分乘子叠加。
基础日费率
该币种基准日保费率（%）。风险越高、历史插针越多，费率越高（故 LAB 显著高于 BTC）。数据来自 dynamic_rates.json（08:00 日报后刷新）。
动态封顶
保费与风险的上限比例：随偿付能力、ATR 波动、1h 插针密度调整（约 10%–75%）。也约束极端行情下的赔付膨胀，属于核心风控旋钮。
实时监控列
1h 振幅
近 1 小时 K 线上的最大插针幅度（%）。旁注「阈 X%」为该币触发理赔的振幅门槛（LAB 约 5%）。带 ⚡ 表示已达或超过阈值。
剩余额度（表中）
与顶部 KPI 相同：全站 24h 剩余额度，各监控币种共享同一池子。
状态 · 未达阈值
实时振幅低于理赔门槛，不赔付。
状态 · 已触发·可赔 / 已触发·拒赔
已达插针条件：可赔表示在额度与偿付规则内可批准；拒赔表示额度用尽、偿付阻断或金额为 0。
状态 · 待行情
尚未拉取交易所 K 线；请运行 monitor 或勾选「拉交易所 live」后刷新。
标注：估算 / 缓存 / 实时
估算 = 仅用费率表+精算引擎；缓存 = 最近一次 monitor；实时 = 本次刷新刚拉行情。
行情源（六所聚合）
与日报一致：对币安 / 欧易 / Gate / Bybit / MEXC / Bitget 分别拉 K 线，取最大 1h 振幅与最多插针次数作为触发依据；表内显示「六所聚合(牵头所)」。赔付额度仍全站共用。
理赔记录
最近理赔记录
由 scripts/wickshield/cli monitor 写入，记录每次触发的决策、原因与拟赔付金额；仅 approved / rejected 会落库。
