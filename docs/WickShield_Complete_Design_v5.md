#### 二、精算模型 3.0

**2.1 多维定价因子体系**
引入历史数据、市场状态、资金池健康度、用户画像及外部风险五大维度，通过加权算法实现千人千面的精准定价。

- **核心因子：** 币种历史插针频率(20%)、市场波动率指数(15%)、当前资金池健康度(15%)、用户历史理赔率(10%)等。

**2.2 动态保费公式**
`日保费率 = 基准费率 × Π(1 + 因子调整系数)`
*注：因子调整系数 = (当前值 - 基准值) / 基准值 × 因子权重*

**2.3 AI增强风险评分**

- **插针预测模型：** 输入历史与市场数据，输出未来24h插针概率风险评分(0-100)。
- **异常检测模型：** 识别用户行为序列中的异常投保模式。
- **资金池压力模型：** 预测赔付流出与LP流入的平衡点，输出压力等级(1-5)。

**2.4 用户分层与折扣体系**
建立从普通用户到机构用户的5级信用分层（保费系数0.7x-1.0x），并结合长期保单、大额投保、早期贡献者等多重折扣叠加规则，激励优质用户留存。

#### 三、AI 异常检测与实时风控

**3.1 智能分级处理策略**
投保请求经AI实时评分后分流：低风险(0-30分)正常处理；中风险(31-60分)增加持仓验证；高风险(61-80分)转入人工/仲裁审核并提高保费；极高风险(81-100分)直接拒绝。

**3.2 核心异常模式拦截**
针对多地址关联投保（图神经网络识别）、投保后立即撤仓（时间序列检测）、跨所套利及机器人批量操作等行为，实施关联限额、自动失效、限流及验证码拦截。

**3.3 实时监控仪表盘**
设定赔付率飙升、单一币种集中投保、新用户异常活跃、LP资金快速流出四大告警阈值，触发自动提费、暂停保单或延长退出等待期等防御措施。

#### 四、偿付能力证明（PoS）

**4.1 链上偿付能力证明**
协议每24小时自动生成链上偿付能力证明（Proof of Solvency），包含以下核心指标：

- **资金池总额：** L1+L2+L3储备金当前余额（每24小时公开）
- **总有效保额：** 所有生效保单的投保金额之和（每24小时公开）
- **偿付能力比率：** 资金池总额 / 总有效保额 × 100%（每24小时公开）
- **L1-L5各层余额：** 逐层展示（每24小时公开）
- **近30天赔付总额与保费收入：** 累计赔付支出与保费收入（每24小时公开）

**4.2 偿付能力预警机制**

- **> 50%（健康）：** 正常运营。
- **30%-50%（关注）：** 暂停新币种上线投保，加强LP招募。
- **20%-30%（警告）：** 暂停新保单，已有保单正常赔付，启动LP紧急注资。
- **10%-20%（危险）：** 赔付比例降至50%，启动再保险协议。
- **< 10%（紧急）：** 暂停所有赔付，启动巨灾债券，触发紧急关停评估。

**4.3 第三方审计**
每季度委托独立审计机构（如Trail of Bits、OpenZeppelin）对偿付能力数据进行链上验证并发布审计报告，确保数据真实可信。

#### 五、压力测试框架

**5.1 测试场景设计**

- **场景A（常规日）：** 日均插针频率下的正常赔付（参考日报历史均值）。
- **场景B（波动日）：** 单一币种发生3-5次插针（参考LAB历史最差日）。
- **场景C（极端日）：** 多币种同时发生插针（模拟市场恐慌）。
- **场景D（灾难周）：** 连续7天高赔付（模拟系统性风险）。

**5.2 测试输出指标与应用**
输出偿付能力比率变化曲线、储备金消耗速度、再保险触发时点及保费调整幅度。测试结果将作为保费率动态调整、LP招募规模规划、巨灾债券发行规模确定及再保险协议条款优化的核心依据。

**5.3 测试频率**
轻量测试（每天，基于前一天实际数据回溯）、完整测试（每周，覆盖所有场景）、极端测试（每月，模拟历史最差行情 + 20%压力加码）。

#### 六、去中心化预言机网络

**6.1 网络架构**
采用“Wick Detector链下引擎 + 去中心化预言机节点”双层架构。链下生成JSON证据与哈希指纹，链上节点进行2/3共识验证后提交至理赔合约。

**6.2 节点激励与约束**

- **奖励：** 成功提交有效事件获保费收入的2%，高准确率加成1.5x，首次发现插针额外奖10%。
- **处罚：** 提交虚假事件罚没50%质押金，连续错误或离线将被移出白名单。

**6.3 数据交叉验证层**
强制要求节点接入至少3家主流交易所API进行交叉比对，结合前后5分钟TWAP均价验证。

#### 七、资金池优化与兑付优先级

**7.1 多层级储备金架构**

- **L1 日常赔付池 (70%)：** 应对常规插针赔付，随时调用。
- **L2 波动储备金 (20%)：** L1不足时启用，应对中等赔付压力。
- **L3 巨灾储备金 (10%)：** L1+L2不足时启用，应对极端行情。
- **L4 再保险与L5 巨灾债券：** 当以上三层耗尽时触发，转移系统性风险。

**7.2 收益优化与对冲**
资金池闲置资金按比例配置于稳定币借贷(Aave)、Uniswap V3流动性提供及代币化国债。同时利用永续合约对冲与期权保护，抵消极端行情下的资金池净流出风险。

**7.3 巨灾债券兑付明确**
明确巨灾债券到期本息兑付资金来源为“协议净盈余”及“L3巨灾储备金的部分投资收益”，严禁动用L1/L2基础赔付本金，保障协议核心偿付能力。

#### 八、紧急关停与有序清算

**8.1 触发条件**
紧急关停仅在以下情况同时满足时启动：偿付能力比率 < 5%（持续超过72小时）、L1-L5全部储备耗尽、再保险机构拒绝续保、DAO投票通过（参与率 > 30%，赞成 > 2/3）。

**8.2 有序清算流程**
触发关停 → 停止新保单 + 停止LP存款 → 所有生效保单到期或赔付后自动终止 → LP按份额比例提取剩余资金 → 投保人未到期保单按剩余天数退还保费 → 用户积分按比例兑换为剩余储备金份额 → 协议进入永久休眠。

**8.3 清算优先级与兜底条款**

1. **已触发未赔付的理赔：** 按触发时间顺序全额赔付。**兜底条款：** 若剩余资金不足以支付优先级1的全额理赔，则所有待赔付订单按剩余资金比例进行分配（即统一执行 Haircut）。
2. **未到期保单：** 按剩余天数比例退还保费。
3. **LP：** 按份额比例提取剩余资金池。
4. **巨灾债券持有者：** 按债券层级优先级偿还本金。
5. **团队成员：** 最后分配（如有剩余）。

#### 九、投保限额与资金池动态联动

**9.1 动态投保限额**
投保上限随资金池健康度实时调整：偿付能力比率 > 50%时单人单次上限500 USDT（单币种24h上限5000 USDT）；30%-50%时分别降至300/3000 USDT；20%-30%时降至100/1000 USDT且暂停新保单；< 20%时完全暂停。

**9.2 大额保单特别处理**
单笔 > 500 USDT的保单需满足：用户信用分 > 800、通过AI异常检测、提交完整持仓证明。大额保单不享受保费折扣。

#### 十、巨灾债券机制

首期规模100万USDT，年化票面利率15-25%，期限1年。触发条件为单周赔付总额超过50万USDT。触发后债券本金转入资金池用于赔付，当期利息停付。设立A级优先层(60%, 15%)、B级中间层(25%, 20%)、C级权益层(15%, 25%)。

#### 十一、争议仲裁与DAO治理

赔付金额与虚假插针争议由AI初审后移交DAO仲裁（48-72小时）；节点作恶举报触发挑战机制与社区投票。仲裁员需持有≥1000 WICK + 信用分>500，每次随机抽取7名，5/7多数决，错误裁决扣除信用分。治理路线遵循“创始团队全权 → 多签委员会 → 混合治理 → 完全DAO”的渐进式路径。

#### 十二、法币出入金与监管科技

**核心合规红线：** 协议智能合约仅处理加密资产。法币出入金须通过持牌第三方支付服务商完成，平台本身不经手任何法币资金。分阶段接入Chainalysis、TRM Labs、Elliptic及Solidus Labs，自动生成SAR与CTR报告。

#### 十三、跨链扩展 v2.0

**13.1 跨链精算模型**
同一币种在不同链上的插针概率不同，必须分开定价。例如SOL在原生链Solana日保费率为0.15%，BTC在BNB Chain因波动更大费率为0.08%（高于Arbitrum的0.05%）。

**13.2 跨链互保防火墙**
各链独立维护LP池，通过再保险协议实现跨链互保。为防止风险传染，单次跨链支援不得超过受援链资金池总规模的30%，且必须要求受援链提供超额抵押物或通过治理投票授权。

**13.3 部署优先级**
Solana(P0) → Arbitrum(P1) → Base(P1) → BNB Chain(P2) → Sui(P2)。

#### 十四、声誉系统与信用SBT

**14.1 信用分链上化与动态管理**
用户信用分达到一定阈值后，可免费铸造对应的声誉SBT（铜牌/银牌/金牌风控师），享受98折至9折不等的保费优惠及优先理赔权。**引入销毁/降级机制：** 当用户信用分跌破当前等级阈值，或经DAO仲裁确认存在恶意行为时，协议有权强制销毁或降级其持有的声誉SBT，并追回已享受的保费折扣差价。

**14.2 跨协议互认**
未来与其他DeFi协议合作，WickShield声誉SBT可作为跨协议的信用凭证（如在借贷协议中作为还款能力的辅助证明）。

#### 十五、产品分级

**15.1 产品线设计**

- **基础版（1.0x基准）：** 仅赔插针事件，面向普通散户。
- **增强版（1.5x）：** 插针 + 预言机故障，面向高级交易者。
- **全能版（2.0x-3.0x）：** 插针 + 预言机故障 + 闪电贷攻击 + 合约漏洞，面向机构用户。

**15.2 全能版理赔触发源**
全能版的理赔触发需依赖多源安全信号（如 Forta Network 警报、官方暂停交易公告等），且由于涉及更复杂的判定，其理赔审核周期（如 72 小时）会长于基础版的插针理赔（秒级/分钟级）。用户可随时升级产品，已支付保费按剩余天数折算抵扣。

#### 十六、代币经济学 v2.0

代币分配涵盖社区激励(40%)、团队与顾问(20%)、生态发展(15%)、早期贡献者追溯(10%)、储备金(10%)、巨灾债券利息储备(5%)。每季度使用“协议净收益”的20%在二级市场回购WICK并销毁，绝不挪用赔付准备金。
